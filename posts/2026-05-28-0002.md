## 二、安装流程

### 1.1 安装 nvm

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
```

含义：从 GitHub 下载 nvm v0.40.4 安装脚本并通过 bash 执行。nvm 安装脚本会自动将配置写入 `~/.zshrc`。

安装完成后需要加载 nvm：

```bash
source ~/.zshrc
```

含义：重新加载 zsh 配置文件，使 nvm 命令可用。**不需要手动执行 export 和 source nvm.sh**，因为安装脚本已经将这些写入 `~/.zshrc`。

验证安装：

```bash
nvm --version
# 输出：0.40.4
```

---

### 1.2 安装 Node.js LTS 版本

```bash
nvm install --lts
```

含义：安装最新的长期支持版 Node.js。当前 LTS 版本为 v24.15.0（代号 Krypton），自带 npm v11.12.1。

**遇到的问题：**

安装时出现提示：

```
Your user's .npmrc file (${HOME}/.npmrc)
has a `globalconfig` and/or a `prefix` setting, which are incompatible with nvm.
Run `nvm use --delete-prefix v24.15.0` to unset it.
```

**问题原因：** `~/.npmrc` 中有 `prefix` 或 `globalconfig` 配置，会强制改变全局包的安装路径，与 nvm 的版本隔离机制冲突。nvm 需要自己控制全局包路径（每个版本独立目录），`prefix` 设置会导致不同版本的全局包混在一起。

**解决方法：**

```bash
nvm use --delete-prefix v24.15.0
```

含义：删除 `.npmrc` 中的 `prefix` 设置，并切换到 v24.15.0。执行后 `~/.npmrc` 文件已被清除（`cat ~/.npmrc` 显示文件不存在）。

---

### 1.3 全局安装 opencode-ai

```bash
npm i -g opencode-ai
```

含义：将 opencode-ai 全局安装到当前 Node.js v24.15.0 环境下。

验证安装：

```bash
opencode --version
# 输出：1.14.19
```

---

### 1.4 查看安装位置

```bash
npm ls -g opencode-ai
# 输出：
# /home/chenbowen/.nvm/versions/node/v24.15.0/lib
# └── opencode-ai@1.14.19
```

含义：opencode-ai 安装在 `/home/chenbowen/.nvm/versions/node/v24.15.0/lib/node_modules/` 下，只存在于 v24.15.0 版本的环境中。

---

### 1.5 撤销别名

```bash
nvm unalias opencode
```

含义：删除之前创建的 `opencode` 别名（别名只是指向 v24.15.0 的快捷方式，没有实际隔离作用，没必要保留）。

## 三、关键概念与问题解答

### 3.1 全局安装不需要隔离

`npm i -g` 安装的是**命令行工具**，不是项目依赖。它：

- 只让 `opencode` 命令出现在 PATH 中
- 不会影响任何项目的 `node_modules/`、`package.json` 或项目依赖
- 其他项目不会引用或依赖 `opencode-ai`
- 类比：全局安装 opencode 就像全局安装 git 或 vim，其他项目不会因为系统有 git 就受到影响

### 3.2 nvm alias 没有必要

`nvm alias opencode v24.15.0` 只是给 v24.15.0 起了一个别名，没有实际隔离作用。因为：

- alias 指向同一个版本，全局包是共享的
- 同一版本号才是真正的隔离边界，alias 不是

### 3.3 全局包 vs 项目依赖

| 安装方式 | 路径 | 作用 |
|---------|------|------|
| `npm i -g xxx` | `~/.nvm/versions/node/v24.15.0/lib/node_modules/xxx` | 全局命令行工具，所有使用此版本的项目共享 |
| `npm i xxx` | `项目目录/node_modules/xxx` | 项目私有依赖，只在该项目内可用 |

同一版本下的全局包是共享的，但这只影响命令行工具，不影响项目依赖。

### 3.4 当前 nvm 版本状态

```bash
nvm ls
```

输出解读：

- `->` 箭头指向当前活跃版本（v24.15.0）
- `system` 是系统自带安装的 Node.js（v22.22.0）
- `lts/*` 指向当前最新 LTS 版本
- 各个 `lts/代号` 行显示历史上所有 LTS 版本代号

## 四、日常使用

### 4.1 使用 opencode

```bash
nvm use v24.15.0   # 切换到 v24.15.0（如果当前不在该版本）
opencode            # 启动 TUI
```

### 4.2 首次使用流程

1. 启动 opencode
2. 执行 `/connect` 配置 API Key
3. 进入项目目录，执行 `/init` 初始化项目
4. 开始提问或让 opencode 修改代码

### 4.3 切换 Node.js 版本

```bash
nvm use v24.15.0    # 切换到 v24.15.0（opencode 在此版本下）
nvm use system      # 切换回系统自带版本
```

### 4.4 查看版本信息

```bash
nvm ls              # 列出本地已安装版本，-> 指向当前活跃版本
nvm current         # 直接输出当前版本号
node -v             # 查看当前 Node.js 版本
npm -v              # 查看当前 npm 版本
opencode --version  # 查看 opencode 版本
```

### 4.5 查看可安装版本

```bash
nvm ls-remote       # 列出所有可安装的远程 Node.js 版本
```

## 五、opencode 使用命令

### 5.1 启动与基本使用

| 命令 | 含义 |
|------|------|
| `opencode` | 启动 TUI（终端交互界面） |
| `opencode --version` | 查看版本号 |
| `opencode -h` | 显示帮助信息 |

### 5.2 认证与 API Key 管理

| 命令 | 含义 |
|------|------|
| `opencode auth login` | 登录并配置 LLM 提供商的 API Key，存储在 `~/.local/share/opencode/auth.json` |
| `opencode auth list` 或 `opencode auth ls` | 查看所有已认证的提供商和 API Key |
| `opencode auth logout` | 登出某个提供商，清除其 API Key |

### 5.3 模型查看

| 命令 | 含义 |
|------|------|
| `opencode models` | 列出所有已配置提供商的可用模型 |
| `opencode models anthropic` | 列出指定提供商（如 anthropic）的可用模型 |
| `opencode models --refresh` | 从 models.dev 重新刷新模型缓存 |

### 5.4 非交互模式运行

```bash
opencode run "Explain how closures work in JavaScript"
```

含义：直接传入提示词，无需启动 TUI，适合脚本和自动化场景。

| 标志 | 含义 |
|------|------|
| `--model provider/model` 或 `-m` | 指定使用的模型 |
| `--agent` | 指定使用的 agent |
| `--file` 或 `-f` | 附带文件到消息 |
| `--share` | 分享该会话 |
| `--continue` 或 `-c` | 继续上次会话 |

### 5.5 会话管理

| 命令 | 含义 |
|------|------|
| `opencode session list` | 列出所有会话 |
| `opencode export [sessionID]` | 将会话数据导出为 JSON |
| `opencode import <file>` | 从 JSON 文件或分享 URL 导入会话 |

### 5.6 TUI 内部命令

在 TUI 交互界面中可使用以下斜杠命令：

| 命令 | 含义 |
|------|------|
| `/init` | 初始化项目，让 opencode 分析项目结构并创建 `AGENTS.md` 文件 |
| `/connect` | 配置 LLM 提供商的 API Key |
| `/undo` | 撤销上次更改 |
| `/redo` | 重做撤销的更改 |
| `/share` | 创建当前会话的分享链接并复制到剪贴板 |

### 5.7 TUI 模式切换

| 操作 | 含义 |
|------|------|
| `Tab` 键 | 切换 Plan 模式和 Build 模式。Plan 模式下 opencode 只提建议不修改文件；Build 模式下可以实际修改文件 |

### 5.8 Agent 管理

| 命令 | 含义 |
|------|------|
| `opencode agent list` | 列出所有可用 agent |
| `opencode agent create` | 创建自定义 agent |

### 5.9 服务与 Web 模式

| 命令 | 含义 |
|------|------|
| `opencode serve` | 启动无界面的 HTTP API 服务器 |
| `opencode web` | 启动带 Web 界面的 HTTP 服务器 |
| `opencode attach [url]` | 将 TUI 附加到已运行的远程 opencode 服务器 |

### 5.10 MCP 管理

| 命令 | 含义 |
|------|------|
| `opencode mcp add` | 添加 MCP 服务器 |
| `opencode mcp list` 或 `opencode mcp ls` | 列出已配置的 MCP 服务器 |

### 5.11 统计与维护

| 命令 | 含义 |
|------|------|
| `opencode stats` | 查看 token 使用量和费用统计 |
| `opencode upgrade` | 升级到最新版本 |
| `opencode uninstall` | 卸载 opencode 并删除相关文件 |
